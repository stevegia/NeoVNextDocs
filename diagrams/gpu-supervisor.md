# Go GPU Supervisor Architecture

> Auto-generated by `scripts/trace_gpu_supervisor.py` — do not edit manually.

The Go GPU Supervisor is a compiled binary that runs on a RunPod GPU pod. It manages
ComfyUI and Python ML handler subprocesses, with intelligent GPU mode switching between
shared and exclusive VRAM allocation. It communicates with the desktop C# API via
Tailscale VPN for file transfer and connects to local PostgreSQL via a socat tunnel.

## 2. GPU Mode State Machine

The supervisor manages two GPU allocation modes to maximize L4 (24 GB) utilization.

```mermaid
stateDiagram-v2
    [*] --> SHARED: Supervisor starts

    SHARED --> DRAINING: Wan 14B job claimed
    DRAINING --> EXCLUSIVE: Analysis handlers stopped,<br/>ComfyUI /free called
    EXCLUSIVE --> RESTORING: Wan job completed
    RESTORING --> SHARED: Analysis handlers restarted

    SHARED --> SHARED: SD/SDXL job or<br/>analysis job processed

    note right of SHARED
        ComfyUI: SD/SDXL workflows (~7 GB)
        + Whisper (~1.5 GB)
        + pyannote (~0.5 GB)
        + InsightFace (~1.5 GB)
        + ViT (~0.5 GB)
        = ~12 GB / 24 GB
    end note

    note right of DRAINING
        1. Stop claiming analysis jobs
        2. Wait for in-flight analysis to finish
        3. SIGTERM analysis handler processes
        4. POST ComfyUI /free (unload SD model)
    end note

    note right of EXCLUSIVE
        ComfyUI: Wan 14B fp8 (~18 GB)
        Only comfyui jobs claimed.
        = ~18 GB / 24 GB
    end note

    note right of RESTORING
        1. POST ComfyUI /free (unload Wan)
        2. Restart analysis subprocesses
        3. Wait for health checks
    end note
```
